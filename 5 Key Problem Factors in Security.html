When an application must accept and process untrusted data that may be malicious....

  1. underdeveloped security awareness
    1. especially with devs, dependencies/sub-dependencies 
  2. Custom development 
    1. after packages, unique code goes in the holes, this introduces unique flaws everytime
  3. Deceptive simplicity with frameworks
    1. making things from scratch quickly = functional 
    2. functional != secure, but many assume that
    3. dependency on frameworks = framework vuln -> every user of it
  4. rapidly evolving threats 
  5. resources and time constraints for development/business 
  6. overextended older tech: old www must peform with new www, designed for old
  7. increasing app functionality demand
