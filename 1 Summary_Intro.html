Fundamental problem: all user input is untrusted
-> shapes sec mechanisms apps use to defend themselves, across apps these are pretty conceptually similar 

Core elements:
  1. user access to app’s data/fxns to prevent users from gaining unauth access
  2. user input to app’s fxns to prevent malformed input ->  undesirable behavior
  3. app functionality during direct attack: suitable defense/offense measures 
  4. admin access to app to monitor activities/config functionality
