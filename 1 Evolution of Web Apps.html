old web: www = web sites
-> information repos w/ static documents
->web browsers: way of retrieving/displaying web docs
*info flow: oneway, server to browser
*auth: usually none, no need to vary info presented based on user 
*vulns: based on web server sw, but server comp didn't lead to much PII (mono users)

modern web: www= web applications
->information: web applications
*info flow: two way, server/browser/server/browser
*auth: needed, much of info processed is highly sensitive 
*vulns: still many, but with PII in the mix now
-each app is varied = different unique vulns
-most developed in house = variety of awareness of sec per dev team
-functionality is based on internal server connections that touch business functions (i.e. bank money transfers)
