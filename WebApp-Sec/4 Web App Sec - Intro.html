SSL protects in transit, but once it gets to the server/client and is dencrypted, it's back open to attack

Key challenge:
-client is outside of application's control
-users on client side can submit artbitrary input to server-side application 
-application must assume all input is potentially malicious, must take steps to sanitize inputs

  • interfere w/ data sent b/t client server - request parameters, cookies, http headers

    • nulls assumptions on how user will interact with app 
  • tools beyond browsers can interact with the API: generate more requests, , etc.
